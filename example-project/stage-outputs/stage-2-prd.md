# Stage 2: Requirements & PRD — 产物

## PRD: VibeTodo 待办清单应用

### 1. 产品概述

| 属性 | 说明 |
|------|------|
| 产品名称 | VibeTodo |
| 定位 | 个人轻量级待办清单工具 |
| 目标用户 | 需要简单任务管理的个人用户 |
| 核心价值 | 打开即用，零学习成本，数据本地保存 |

### 2. 功能需求

#### F1: 添加任务 [P0]

| 字段 | 类型 | 约束 | 默认值 |
|------|------|------|--------|
| 任务标题 | 文本 | 必填，1-200 字符，自动去除首尾空格 | — |
| 优先级 | 枚举 | 三选一：高(红)/中(橙)/低(蓝) | 中 |

**交互流程**:
1. 用户输入标题文本
2. 选择优先级（默认"中"）
3. 点击"添加"按钮或按 Enter 键
4. 任务出现在列表顶部
5. 输入框清空，焦点保持在输入框

**校验规则**:
- 空内容 → 不添加，输入框抖动提示
- 纯空格 → 同空内容处理
- 超过 200 字符 → 截断并提示

#### F2: 标记完成/取消完成 [P0]

**交互**: 点击任务项左侧的复选框或任务文字区域
**状态变化**:
- 未完成 → 已完成：文字加删除线，透明度降低，completedAt 记录时间
- 已完成 → 未完成：恢复正常显示，清除 completedAt

#### F3: 删除任务 [P0]

**交互**: 点击任务项右侧的删除按钮
**行为**:
- 直接删除（无需确认弹窗 —— 数据有 localStorage 可恢复）
- 删除时有淡出动画效果（200ms）
- 如果删除的是最后一个某优先级的任务，筛选状态不变

#### F4: 优先级筛选 [P1]

**UI 形式**: 顶部 Tab 栏：全部 | 高 | 中 | 低
**行为**:
- 默认显示"全部"
- 点击某个优先级 → 只显示该优先级的任务
- 当前选中 tab 高亮显示
- 任务数为 0 时显示空状态提示文案

#### F5: 任务计数 [P1]

**显示位置**: 页面底部或 header 区域
**格式**: "共 X 项，已完成 Y 项"
**实时更新**: 每次增删改操作后重新计算

#### F6: 数据持久化 [P0]

**存储方式**: localStorage
**Key**: `vibetodo-todos`
**触发时机**:
- 添加任务后立即保存
- 标记完成/取消后立即保存
- 删除任务后立即保存
**容错**:
- localStorage 不可用时 → 显示警告条，降级为内存模式（刷新丢失）
- 数据损坏时（JSON 解析失败） → 清空并从空白开始

### 3. 非功能需求

| ID | 需求 | 指标 |
|----|------|------|
| NF1 | 性能 | 首次加载 < 1s，操作响应 < 100ms |
| NF2 | 兼容性 | Chrome/Firefox/Safari/Edge 最新两版 + iOS Safari / Android Chrome |
| NF3 | 可访问性 | 语义化 HTML，键盘可操作（Tab 导航，Enter/Space 触发） |
| NF4 | 响应式 | 320px (手机 SE) ~ 1920px (桌面) 全适配 |
| NF5 | 安全 | 无 XSS 风险，所有用户内容使用 textContent 渲染 |
| NF6 | 离线可用 | 纯前端应用，天然离线（无需网络） |

### 4. 信息架构

```
VibeTodo Page
├── Header
│   ├── Logo/Title ("VibeTodo")
│   └── Task Counter ("共 X 项，已完成 Y 项")
├── Input Section
│   ├── Text Input (placeholder: "添加新任务...")
│   ├── Priority Selector (高/中/低)
│   └── Add Button (+)
├── Filter Tabs
│   ├── Tab: 全部 (count)
│   ├── Tab: 高 (count)
│   ├── Tab: 中 (count)
│   └── Tab: 低 (count)
├── Task List
│   └── Task Item (repeat)
│       ├── Checkbox (toggle complete)
│       ├── Title text
│       ├── Priority badge
│       └── Delete button
└── Footer
    └── Empty State (when list is empty)
        └── "还没有任务，添加一个吧！"
```

### 5. 用户故事

**US-1**: 作为用户，我想快速添加一个待办任务，以便记录我需要做的事情。
**US-2**: 作为用户，我想标记任务已完成，以便跟踪我的进度。
**US-3**: 作为用户，我想删除不再需要的任务，以保持列表整洁。
**US-4**: 作为用户，我想按优先级查看任务，以便专注于重要的事情。

- [ ] 可以切换任务的完成状态
- [ ] 可以删除任务

- [ ] 显示正确的任务计数
- [ ] 刷新页面后数据保留

- [ ] 无控制台错误
- [ ] 空输入无法添加
- [ ] XSS 安全（innerHTML 未用于用户内容）
