# Stage 1: Ideation & Scoping — 产物

## 用户原始需求

> "我想做一个待办清单 App，可以添加任务、标记完成、删除任务，最好能按优先级分类，数据保存在浏览器里不丢失。"

## AI 引导式提问与回答

### Q1: 这个待办清单是给自己用还是多用户？

**A**: 自己用就行，单用户。

**→ 决策**: 不需要登录/注册系统。不需要后端。纯前端应用。这大大简化了范围。

### Q2: 主要在什么设备上使用？

**A**: 手机和电脑都要能用。

**→ 决策**: 必须做响应式设计（Stage 4 重点）。

### Q3: "按优先级分类" 具体是什么意思？是筛选还是分组展示？

**A**: 能筛选就好，不用复杂的分组。

**→ 决策**: 用 filter tabs 实现，不需要拖拽分组。

### Q4: 除了你提到的功能，还有其他想要的吗？（提醒、截止日期、标签…）

**A**: 不用了，先做最核心的功能。

**→ 决策**: 严格执行 MVP 思维。不加入截止日期、标签、提醒等扩展功能。

## Scope 定义

### MVP 核心范围（本次实现）

| # | 功能 | 描述 | 优先级 |
|---|------|------|--------|
| F1 | 添加任务 | 输入标题 + 选择优先级 → 加入列表 | P0 |
| F2 | 标记完成 | 点击切换完成状态，视觉上区分 | P0 |
| F3 | 删除任务 | 移除不需要的任务 | P0 |
| F4 | 优先级筛选 | 按 高/中/低/全部 筛选显示 | P1 |
| F5 | 任务计数 | 显示总数和已完成数 | P1 |
| F6 | 数据持久化 | 刷新页面不丢失数据 | P0 |

### 明确排除（不在本次范围）

| 功能 | 排除原因 | 未来考虑 |
|------|---------|----------|
| 用户登录/注册 | 单用户场景不需要 | V2.0 |
| 截止日期/提醒 | 用户明确说先做核心的 | V1.1 |
| 标签/分类 | 用户说筛选就够了 | V1.1 |
| 拖拽排序 | 复杂度高，非 MVP | V1.2 |
| 分享/协作 | 单用户场景 | V2.0 |
| 暗色模式 | 锦上添花，非核心 | V1.1 |
| 导出/导入 | 低频需求 | V2.0 |

## 技术方向预判

基于以上范围：

| 维度 | 选择 | 理由 |
|------|------|------|
| 前端框架 | **无框架 (Vanilla JS)** | 项目极简，无需 React/Vue 的复杂度 |
| 构建工具 | **无构建步骤** | 即开即用，降低门槛 |
| 数据存储 | **localStorage** | 单用户、小数据量、浏览器原生支持 |
| CSS 方案 | **原生 CSS + 自定义属性** | 展示设计系统思维，无需 Tailwind |
| 图标 | **Unicode 字符或 CSS 绘制** | 零外部依赖 |
