# Module B: Rule Package Version Management

## Problem

As projects evolve through the 8-stage workflow, rules accumulate in multiple files:

```
CLAUDE.md          (Stage 0 generated, updated during Stages 3/6/7)
.cursorrules       (Stage 0 generated)
INITIAL.md         (Stage 0 template)
security-checklist.md (reference)
prompt-templates.md (reference)
self-check.md      (reference)
```

Issues that arise:
- Rules become stale as project evolves
- No version history of rule changes
- Hard to sync across different AI assistants (Claude Code + Cursor + Copilot + WorkBuddy)
- No easy rollback when a change causes regressions

## Solution: Rule Pack System

A **Rule Pack** is a versioned bundle of project-specific AI instructions that can be synced across assistants.

### Pack Structure

```
.rulebook/
├── pack.toml              # Manifest (version, metadata, dependencies)
├── core-rules.md          # Consolidated core rules (merged from CLAUDE.md + .cursorrules)
├── security-rules.md      # Security-specific rules (from security-checklist.md)
├── style-guide.md         # Coding style conventions
├── patterns.md            # Discovered code patterns (from examples/)
├── changelog.md           # Version history of rule changes
└── v1/
    ├── core-rules.md      # Snapshot of v1 rules (for rollback)
    └── manifest.json      # v1 checksums and timestamps
```

### pack.toml Manifest Format

```toml
[pack]
name = "my-project"
version = "1.2.0"
created_at = "2026-06-14T10:00:00Z"
updated_at = "2026-06-14T14:00:00Z"
author = "vibe-coding-flow"

[targets]
claude_code = true        # Generate CLAUDE.md format
cursor = true             # Generate .cursorrules format
copilot = false           # Future: generate .github/copilot-instructions.md
workbuddy = true          # Our native format

[rules]
core = "core-rules.md"
security = "security-rules.md"
style = "style-guide.md"
patterns = "patterns.md"

[changelog]
# Auto-generated entries when rules are modified

[[changelog.entry]]
version = "1.2.0"
timestamp = "2026-06-14T14:00:00Z"
stage = 7
change = "Added textContent safety rule from Prompt Debt auto-codification"
reason = "L4 detected repeated 'use textContent' pattern > 3 times"

[[changelog.entry]]
version = "1.1.0"
timestamp = "2026-06-14T12:30:00Z"
stage = 6
change = "Updated tech stack from React to vanilla JS per Stage 3 pivot"
reason = "Architecture decision changed"

[version_history]
current = "1.2.0"
previous = ["1.1.0", "1.0.0"]
rollback_support = true  # Can revert to any previous version
```

### Validation Script

Use the included validation script to generate and validate packs:

```bash
# Generate a template pack.toml
python references/scripts/validate-pack.py generate .rulebook my-project

# Validate an existing pack
python references/scripts/validate-pack.py validate .rulebook/pack.toml

# Check consistency between pack and actual files
python references/scripts/validate-pack.py check .
```

See [scripts/validate-pack.py](scripts/validate-pack.py) for the full implementation.

### Workflow Integration

| Stage | Rule Pack Action |
|-------|-----------------|
| **Stage 0** | Initialize `v1.0` pack with initial rules |
| **Stage 3** | Update `core-rules.md` with confirmed tech stack → bump to `v1.1` |
| **Stage 6** | On auto-fix or pattern discovery → update `patterns.md` → bump minor version |
| **Stage 7** | Final consolidation → bump patch version → generate target format files |

### Sync Across Assistants

```bash
# Generate CLAUDE.md from rule pack
rulebook export --target claude_code --output CLAUDE.md

# Generate .cursorrules from rule pack
rulebook export --target cursor --output .cursorrules

# Check if local files are out of sync with pack
rulebook status  # Shows: "Pack v1.2.0 but CLAUDE.md is at v1.1.0 — RUN EXPORT"
```

### Rollback Example

```bash
# Something went wrong after rule update? Rollback:
rulebook rollback --to 1.1.0
# Restores all rule files to v1.1.0 state
# Records rollback in changelog.md
```

### Minimal Viable Implementation

For now (without building a full `rulebook` CLI tool), implement as:

1. A `.rulebook/pack.toml` file maintained manually (but with clear format)
2. A changelog section appended to MEMORY.md on each significant rule change
3. Version annotations in CLAUDE.md header (`<!-- Generated by vibe-coding-flow v3, Rule Pack v1.2.0 -->`)
4. Manual rollback by restoring from `.rulebook/v{N}/` snapshots

This gives 80% of the value with 20% of the effort.
